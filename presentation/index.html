C'est quoi l'effet waouh?

L’effet « waouh » ou « wow » désigne le fait qu’un produit ou service, une expérience ou une campagne publicitaire puisse déclencher chez les consommateurs un effet de surprise, d’admiration ou d’appréciation pouvant notamment se traduire par l’expression « waouh! » ou « wow! ».

L’effet wow est généralement obtenu par :

  • une caractéristique forte et innovante du produit ou service
  • un message ou format publicitaire particulièrement impactant
  • une théatralisation du point de vente particulièrement spectaculaire
Dans le domaine des caractéristiques produit, on a ainsi beaucoup parlé d’effet « wow » pour les consommateurs découvrant pour la première fois les effets de l’écran 3D de la console Nitendo 3DS lors de son lancement en 2011.

Selon sa nature et son contexte d’usage (produit, magasin, publicité, etc.), l’effet waouh peut correspondre à différents objectifs marketing et commerciaux qui peuvent s’avérer complémentaire :

  • favoriser l’achat classique ou d’impulsion
  • diminuer la sensibilité au prix et la rationalité consommateur
  • favoriser l’expérience de marque, l’image et la mémorisation (top of mind)
  • favoriser la recommandation et les retombées sociales (instagrammabilité)

Les applications natives VS les applications Web

D'un côté, nous avons les sites Web accessibles depuis un navigateur …

  • Le site web mobile est un second site qui prend le relais du site principal lorsque l’utilisateur est sur une tablette ou sur un smartphone
  • Un site Internet « Responsive » est un site conçu avec un Design Responsive, c’est-à-dire dont le design s’adapte à tous les supports
  • Une application Web (Webapp) est un site Internet développé pour les mobiles. Par extension, elle désigne aussi une web application mobile
supports : ordinateur, smartphone et tablette

… des applications disponibles depuis une plateforme de téléchargement :

  • développées pour un système d’exploitation spécifique
  • téléchargées via des plateformes dédiées, par exemple, App Store ou Play Store
Il existe aussi l’ application hybride qui est une version hybride d’une application web et d’une application native. Elle s’exécute via un store et s’adapte à tous les systèmes d’exploitation.
Système d'exploitation : iOs, Android. Il faut bien distinguer les interfaces accessibles depuis un navigateur Web (site mobile, site Responsive Design, Webapp) des interfaces accessibles depuis un store (application native, application hybride).

C’est quoi une Progressive Web APP (PWA) ?

Amélioration progressive

A l'inverse du concept de dégradation gracieuse, le principe d’ amélioration progressive ("progressive enhancement" en anglais) est une stratégie de conception web en couches successives, qui permet à chacun d’accéder au contenu et aux fonctionnalités de base d’une page web en utilisant n’importe quel navigateur, tout en offrant une version améliorée aux utilisateurs disposant de navigateurs plus récents ou plus évolués.

les App Shell dans le cas des single-page applications (SPA)

  • utilisé pour obtenir du code minimal HTML, CSS et JavaScript requis pour charger l'écran
  • Le contenu dynamique est chargé pour chaque page en utilisant JavaScript
  • la mise en cache permet d'assurer un chargement plus performant lors de visites répétées
Cela signifie que l'application n'est pas chargé à chaque visite de l'utilisateur. Seul les données dynamiques nécessaire est chargées.
En d'autres termes, l'App Shell est similaire au bundle de code que vous publieriez sur un app store lors de la création d'une application native. Il est le squelette de votre interface utilisateur et les composants de base nécessaires pour démarrer l'application, mais ne contenant pas de données.

les avantages d'une architecture App shell

  • performances fiables et rapides: Les visites sont extrêmement rapides. Les éléments statiques et l'interface utilisateur sont mis en cache lors de la première visite, et peuvent être rechargés lorsque c'est nécessaire.
  • Interactions natives: Les App Shell permettent de créer des applications se comportant comme des applications natives avec le support hors connection.
  • Utilisation économique des données: Les App Shell permettent une utilisation minimale des données puisque seule les données dynamiques seront téléchargées à chaque visite.

les services worker

L'architecture App Shell s’appuie sur les Service Workers, des scripts qui fonctionnent en parallèle de la page Web. Ils permettent le mode hors-ligne, l’envoi de notifications ou la mise à jour des contenus en arrière-plan.
