Applications natives

Le site web mobile

Un second site qui prend le relais du site principal lorsque l’utilisateur est sur une tablette ou sur un smartphone

-> Meilleure rapidité, fiabilité et dotée d’une meilleure réactivité ainsi qu’une résolution supérieure ce qui assure une meilleure expérience utilisateur. -> Elle permet un accès plus facile à toutes les fonctionnalités du téléphone, de l’accéléromètre en passant par la caméra et même le micro. -> Les notifications push, uniquement disponibles sur les apps native. Ces notifications vous permettent d’alerter vos utilisateurs et d’attirer leur attention chaque fois que vous le souhaitez, que ce soit pour du nouveau contenu ou une offre promotionnelle. -> Ne requiert pas forcément internet pour fonctionner, ce qui est un réel avantage. Même en 2017, il existe encore des zones très peu couvertes par le réseau internet, et permettre à ses utilisateurs d’accéder à l’app sans connexion web est un très gros point fort à ne pas négliger. -> spécifique à une plateforme

Applications natives

  • développées pour un système d’exploitation spécifique
  • téléchargées via des plateformes dédiées (Store)
Système d'exploitation : iOs, Android. Il faut bien distinguer les interfaces accessibles depuis un navigateur Web (site mobile, site Responsive Design, Webapp) des interfaces accessibles depuis un store (application native, application hybride).

L'application hybride

  • à la fois une application web et une application native
  • s'execute via un store
  • adaptée à tous les systèmes d’exploitation
-> plus facile et plus rapide à développer qu’une app native -> les performances de l’application sont moins bonnes et moins stables -> ionic, react native

C’est quoi une Progressive Web APP (PWA) ?

Amélioration progressive

c'est stratégie de conception web qui permet d’accéder au contenu et aux fonctionnalités de base d’une page web, tout en offrant une version améliorée.
https://www.alsacreations.com

App Shell

  • conteneur minimal pour executer du HTML, CSS et JavaScript
  • mise à jour des données via un thread JavaScript dédié
  • la mise en cache des resources statiques
https://developers.google.com
En d'autres termes, l'App Shell est similaire au bundle de code que vous publieriez sur un app store lors de la création d'une application native. Il est le squelette de votre interface utilisateur et les composants de base nécessaires pour démarrer l'application.
